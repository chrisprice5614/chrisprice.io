ASCII 10 or ASCII 13?

By Chris Price
Link copied to clipboard!

Parsing Problem

If you're parsing through a ton of files like I often do, you've probably run into a notorious character making a getting to the next line a hastle. One would assume that looking for \n would be enough forcatching a new line, but then you run into another odd character that seems to also create a new line. ASCII 13 is the pesky piece of data that's summoning this new line, but what exactly is it?

In old DOS terminals, ASCII 10, also known as new line, line feed or \n, would shift the upcoming text down to the next line, but not shift the text back to the start of the column.

That's where ASCII 13 comes in

ASCII 13 is known as the carriage return, named as such because it moves the line back to the left. The name comes from typewriters that would finish a line, move down and then move to the left to start a new line. Carriage returns are often used as \r when handling text files with code. In older computer typing software, new lines would be created by making a line feed, followed by a carriage return, just as a typewriter would do.

In most of today's text files, carriage returns are often thrown out and not worried about, only using the line feed. This is why if you try to load newer text files into older text readers the format will be very off. Carriage returns still show up in quite a few formats though, such as excel files. If one were to convert an excel file into a text file to parse the data, they'd run into the pesky carriage return.

Dealing with ASCII 13

Often times when I'm parsing through text, I throw away the carriage return. I use GameMaker Studio 2.3 for doing most of my programming work, so it's easy to get rid of it with string_replace_all(str,ansi_char(13),"");

When writing a text file via code I include it just in case a reader doesn't handle text files very well. When programming, it's always very important to remember accessability, backwards compatibility, and sustainability. You want your programs to run easily on old systems, current systems and future systems. Keeping the carriage return will ensure that your text file will be easily read by most applications.
